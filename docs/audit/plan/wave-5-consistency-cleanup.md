# Wave 5 — Consistency, naming & dead-code cleanup — Remediation Plan

> Scope: this wave is the mechanical-hygiene and catch-all pass that brings Kiri's own code to a single, predictable shape so the whole tree reads as one author. It unifies the module-file convention, the `Result` alias, and the `#[async_trait]` spelling; deletes genuinely-dead code and re-gates test-only helpers; trims a speculative full-CRUD memory-port surface (YAGNI); single-sources scattered magic strings/numbers (the `"function"` tool kind, the 30s/64KiB tool limits, the `"(sem título)"` label, the wheel-scroll step, the spinner glyph); aligns model-facing protocol text on English; converges DTO/enum-conversion naming; makes the re-export-root/module `//!` convention uniform; adds every missing error-justification comment the contract requires; and removes a handful of silent no-ops on user intent. Every change is behavior-preserving or test-locked, fully gated by `cargo fmt --check -> clippy -D warnings -> build -> test`. It ships as a unit because it touches almost every file once for cross-cutting consistency, and doing it in one pass avoids re-touching the same files across several PRs.

## Remediation steps (ordered to de-risk within the wave)

### Step 1 — Unify the module-file convention on `<name>.rs` and record it (CONS-01 / STRUCT-06)
- **Problem** — `memory`/`session`/`sync` declare their layers from `mod.rs` while every other module uses the `<name>.rs` sibling style; a reader cannot predict where a submodule list lives. Pure drift, no functional reason.
- **Approach** — For each of the 10 directories, `git mv <dir>/mod.rs <dir>.rs` so e.g. `memory/application/mod.rs` becomes `memory/application.rs` sitting beside `memory/application/`. The 10 moves: `memory/{application,domain,infrastructure,infrastructure/tools}`, `session/{application,domain,infrastructure}`, `sync/{application,domain,infrastructure}`. No code inside changes — the file content (`pub mod …`) is identical; only the path moves, which Rust resolves identically. Add one line to CLAUDE.md "Project rules / gotchas": "Module convention: declare submodules from the sibling `<name>.rs`, never an inner `mod.rs` (edition-2024 default, matches the whole tree)."
- **Files touched** — moves only: `src/modules/memory/application/mod.rs -> application.rs`, `…/domain/mod.rs -> domain.rs`, `…/infrastructure/mod.rs -> infrastructure.rs`, `…/infrastructure/tools/mod.rs -> tools.rs`; the same for `session/{application,domain,infrastructure}` and `sync/{application,domain,infrastructure}`; modified `CLAUDE.md`.
- **Tests first** — None new (no behavior). The gate is the regression contract: `cargo build` + `cargo test` must still compile/pass, proving the module paths resolve. Run `cargo fmt --check` to confirm no formatting drift.
- **ADR** — none (the audit explicitly asks for a CLAUDE.md note, not an ADR).
- **Risk & dependencies** — Near-zero blast radius (path-only). Do this first so every later edit lands in the final file shape and avoids a rename-then-edit churn. No cross-wave dependency.

### Step 2 — Standardize the `#[async_trait]` spelling tree-wide (CONS-02)
- **Problem** — Three spellings coexist: `#[async_trait::async_trait(?Send)]`, the bare-import `use async_trait::async_trait;` + `#[async_trait]` / `#[async_trait(?Send)]` (the memory/session/sync cluster, **including two `#[cfg(test)]` modules**), and `#[async_trait::async_trait]`. The `?Send` vs `Send` distinction is real; the import-vs-path split for identical semantics is drift.
- **Approach** — There are **15** files carrying `use async_trait::async_trait;` (verified: `grep -rl 'use async_trait::async_trait;' src/` returns 15 — the original plan's "13" omitted `memory/application/distill.rs` and `sync/application/sync_service.rs`, both of which import it inside their `#[cfg(test)]` module). In each, delete that import and convert each attribute to the path-qualified form, **preserving the per-attribute `Send`/`?Send` variant** — do not apply a blanket "all are Send" rule:
  - Production traits in the cluster are `Send` → `#[async_trait::async_trait]`.
  - `memory/application/distill.rs`: the test-only `ScriptedProvider` impls `CompletionProvider` with `#[async_trait(?Send)]` (the engine port is single-threaded) → convert to `#[async_trait::async_trait(?Send)]` and keep `(?Send)`.
  - `sync/application/sync_service.rs`: the test-only `FakeGit` impls `Git` with `#[async_trait]` (Send) → convert to `#[async_trait::async_trait]`.
  Leave the already-path-qualified `?Send` engine ports unchanged. Net rule, single-sourced in CLAUDE.md alongside Step 1's note: "always path-qualified `#[async_trait::async_trait]`; add `(?Send)` only for the single-threaded engine ports (and any test double impl'ing one)." Verify no remaining `use async_trait::async_trait;` with grep.
- **Files touched** — modified (15): `memory/application/{memory_port,project_memory,project_store,shared_memory,shared_store,distill}.rs`, `memory/infrastructure/{file_project_memory,file_project_store,sqlite_shared_memory,sqlite_shared_store}.rs`, `session/application/session_store.rs`, `session/infrastructure/sqlite_session_store.rs`, `sync/application/{git,sync_service}.rs`, `sync/infrastructure/git_cli.rs`; `CLAUDE.md`.
- **Tests first** — None new (attribute spelling is identical macro expansion; `(?Send)` is preserved so the test doubles still compile). `cargo build`/`cargo test` is the regression lock; `grep -rl 'use async_trait::async_trait;' src/` returning **empty** is the verification (this is exactly the wave's DoD #2 gate — converting all 15, not 13, is what makes the gate pass).
- **ADR** — none.
- **Risk** — Mechanical, compiler-checked. The only non-mechanical subtlety is the `(?Send)` preservation in `distill.rs`, called out above. No dependency.

### Step 3 — Settle the `Result`-alias policy on one explicit kernel alias (CONS-03 / DUP-10)
- **Problem** — `type Result<T> = std::result::Result<T, AgentError>;` is re-declared in 15 files (all under memory/session/sync) and shadows `std::result::Result`, while agent/provider/tui/shared spell `Result<…, AgentError>` in full — and `sync` does both. A reader skimming a memory file sees a bare `Result<T>` whose error type is implicit.
- **Approach** — Add `pub type AgentResult<T> = Result<T, AgentError>;` to `src/shared/kernel/error.rs` (next to `AgentError`), with a one-line doc. Delete the 15 local `type Result<T>` shadows; in each file `use crate::shared::kernel::error::AgentResult;` and replace `Result<T>` signatures with `AgentResult<T>`. This keeps the error type named at every signature and removes the std shadow. Make `sync` internally consistent (git.rs's full spelling becomes `AgentResult<GitOutput>`). Record the policy in CLAUDE.md ("fallible port/adapter signatures use `AgentResult<T>` from the kernel; `anyhow::Result` only at the binary edge").
- **Files touched** — modified: `shared/kernel/error.rs` (add alias) and the 15 files listed under DUP-10 (`memory/application/{distill,memory_port,project_memory,project_store,shared_memory,shared_store}.rs`, `memory/infrastructure/{docs_library,file_project_memory,file_project_store,sqlite_shared_memory,sqlite_shared_store}.rs`, `session/application/session_store.rs`, `session/infrastructure/sqlite_session_store.rs`, `sync/application/sync_service.rs`, `sync/infrastructure/memory_ndjson.rs`); `CLAUDE.md`.
- **Tests first** — None new (type-alias substitution, compiler-verified). Regression: `cargo build` + grep asserting zero `type Result<T>` remain.
- **ADR** — none.
- **Risk** — Mechanical but touches 15 files; compiler catches any miss. **Cross-wave:** DUP-07 (typed `AgentError::memory/session/sync` constructors) edits the same files — sequence this alias decision first so DUP-07 builds on it.

### Step 5 — Trim the speculative memory-port surface and collapse the double-port tier (MEM-02 / BUILD-02, MEM-03, MEM-01, DUP-11)
- **Problem** — `ProjectMemory`/`SharedMemory` carry a trait-wide `#[allow(dead_code)]` ("reserved for the future UI"), keeping a large full-CRUD surface alive with no runtime caller (`delete`/`count`/`count_by_project`/`list_by_*` are tests-only). Memory persistence is modeled as two parallel port tiers (`*Memory` full-CRUD vs `*Store` use-cases) where the Store adapter just delegates — redundant indirection. The Project/Shared pairs are byte-identical except `Shared` adds `list_by_project`, and `save_embedding`/`embedded_candidates` defaults are copy-pasted into both Store traits. Four hand-rolled in-memory test doubles duplicate the same bodies, and a `sandbox()` test fixture is triplicated.
- **Approach** —
  - **Trim (YAGNI):** delete from `ProjectMemory`/`SharedMemory` and their adapters the methods with no production caller (`delete`, `count`, `count_by_project`, plus any `list_by_*` not consumed by the runtime — verify each with grep first; the audit confirms the runtime uses only `init`/`save`/`search`/`list_by_kind`/`list_by_tag`/`load`, and `sync` uses `SharedMemory::{list,load,save}`). Remove the trait-wide `#[allow(dead_code)]`. Keep `list_by_kind`/`list_by_tag` (live).
  - **Collapse the tier:** the runtime wires `ProjectStore`/`SharedStore`; the `*Memory` traits are the indirection. Decide one: keep the `*Store` ports the runtime consumes and fold the embedding-persistence + remaining methods onto them, deleting the now-empty `*Memory` tier — OR vice-versa. Plan recommends keeping `*Store` (what `wire` injects) and removing the `*Memory` traits, moving `save_embedding`/`embedded_candidates` to a single shared base trait.
  - **Factor the pairs (MEM-03):** introduce `MemoryStore: Send + Sync` with `save`/`search`/`list_by_kind`/`list_by_tag`/`is_available` + the two embedding defaults (defined once), and `SharedStore: MemoryStore` adding `list_by_project`.
